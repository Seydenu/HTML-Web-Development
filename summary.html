Firstly, everything that,s displayed in CSS is a box.

Content & Sizing

Boxes have different brhaviors based on theis display value, their set dimensions and the content that lives within them.

You can control this by using "extrinsic sizing" or you can let the browser make decision for you using "instrinsic sizing."

Using instrinsic sizing can allow the browser to well place your contents on the webpage as it should best fit.

When the element is using the extrinsic sizing, there's a limit of how much content you can add before it overflows out of it element's bounds, but this isn't the case with when instrinsic sizing is toggled on.

The areas of the box model

Boxes are made up of: Margin box, border box, padding box, & content box.

  1. The Content Box - Is the area the content lives in. This content can control the size of its parents, so is most likely the most variable sized area.
  2. The Padding - Sorrounds the content box and it's the space created by the padding property.
  3. The Border Box - Sorrounds the padding box and its space is occupied by the border value.
  4. The Margin box - Is the space around your box, defined by the margin rule on your box.

Controlling the Box Model

It's the user agent style where the box's default "display" is set, for example: If we're in a normal flow, a "div" element's default "display" value is "block", a "li" has default display value of "list-items", and "span" has a default display value of "inline".
